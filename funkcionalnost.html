Kako Microsoft Azure radi?

Azure je javna cloud platforma kompanije Microsoft. Azure pruža veliki set usluga, uključujući platformu kao servis Azure:(PaaS), infrastrukturu kao servis Azure:(IaaS) i softver kao servis Azure:(SaaS). Međutim, šta tačno predstavlja Azure i kako funkcioniše?

Azure kao i ostale cloud platforme oslanja se na tehnologiju poznatu kao virtualizacija. Većina računarskog hardvera može biti emulirana pomoću softvera. Računarski hardver jednostavno predstavlja skup instrukcija koje su trajno ili polutrajno kodirane u silikonu. Slojevi emulacije se koriste kako bi se mapirale softverske instrukcije na hardverske instrukcije. Ovi slojevi omogućavaju virtualiziranom hardveru da izvršava softver kao stvarni hardver.

Esencijalno, cloud predstavlja skup fizičkih servera u jednom ili više datacentru. Datacentri izvršavaju virtualizirani hardver za korisnike. Kako, onda, cloud istovremeno stvara, pokreće, zaustavlja i briše milione instanci virtualiziranog hardvera za milione korisnika?

Da bismo razumeli servere, pogledajmo arhitekturu hardvera u datacentru. Unutar svakog datacentra nalazi se skup servera smeštenih u rack-ove. Svaki server rack sadrži mnogo server blades-a i mrežni switch. Ovi pružaju mrežnu povezanost i jedinicu za distribuciju napajanja (PDU), koja obezbedjuje električnu energiju. Rack-ovi se ponekad grupišu u veće jedinice poznate kao klasteri.

Kod Microsoft Azure server rack-ovi ili klasteri, odabrani su da pokreću virtualizirane instance hardvera za korisnike. Medjutim, neki serveri pokreću softver za upravljanje cloud-om, poznat i kao fabric controller. Fabric controller je distribuirana aplikacija sa mnogo odgovornosti. On alocira usluge, prati zdravlje servera i usluge koje se na njemu izvršavaju i obnavlja server kada dodje do kvara.

Svaka instanca fabric controller-a povezana je sa drugim setom servera koji pokreću softver za cloud orkestraciju, obično poznat kao front end. Front end hostuje web servise, RESTful APIs i interne Azure baze podataka, koje se koriste za sve funkcije u cloud-u.

Na primer, front end hostuje usluge koje obradjuju zahteve korisnika. Zahtevi alociraju Azure resurse i usluge poput Azure Virtual Machines i Azure Cosmos DB. Prvo, front end validira i proverava da li je korisnik ovlašćen za alokaciju traženih resursa. Ako jeste, front end proverava bazu podataka kako bi pronašao server rack sa dovoljnim kapacitetom, što primorava fabric controller da alocira resurse.

Azure je ogromna kolekcija servera i mrežnog hardvera, koji pokreće kompeksan set distribuiranih aplikacija. Ove aplikacije orkestriraju konfiguraciju i operaciju virtualiziranog hardvera i softvera na tim serverima. Orkestracija ovih servera je ono što čini Azure tako moćnim. Sa Azure-om, korisnici ne moraju održavati i nadogradjivati svoj hardver, jer azure to radi iza scene.
